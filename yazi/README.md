# yazi

[Yazi](https://github.com/sxyazi/yazi) TUI file manager configuration.

## Dependencies

- `yazi`
- `fzf`, `fd`, `ripgrep`, `zoxide` (optional but used by plugins)

## Plugins

Plugins are managed via `package.toml`. After cloning, install them:

```sh
ya pack -i
```

Included plugins:
- `full-border.yazi` — full border around the UI panels
- `mime-ext.yazi` — extended MIME type detection
- `smart-enter.yazi` — opens files or enters directories with one key

## Flavors

`flavors/` contains color themes. `flavors/flavors` is a git submodule pointing to the official [yazi-rs/flavors](https://github.com/yazi-rs/flavors) repository. After cloning the dotfiles repo:

```sh
git submodule update --init --recursive
```

`flavors/matugen.yazi/` is auto-generated by matugen/DMS — don't edit it directly.

## The `y` shell function

`fish/config.fish` defines a `y` wrapper function that launches yazi and changes the shell's working directory to wherever you navigated when you exit. This requires the `--cwd-file` flag supported by yazi.
