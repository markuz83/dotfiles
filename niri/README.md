# niri

[Niri](https://github.com/YaLTeR/niri) scrollable-tiling Wayland compositor configuration.

## Dependencies

- `niri`
- DMS (DankMaterialShell)

## Important: most config lives in `dms/`

`config.kdl` includes files from `./dms/` — these are **auto-generated by DMS** and are not tracked here. DMS generates keybinds, colors, outputs, cursor, window rules, alt-tab, and wallpaper blur config.

Only `config.kdl`, `layout-default.kdl`, and `animations.kdl` are manually maintained.

## Config format

Niri uses [KDL](https://kdl.dev) (not TOML/JSON). `/-` comments out the next node (used for the disabled `environment {}` block — those env vars are set via `environment.d` instead).

## Session startup

Unlike Hyprland, niri starts as a systemd service (`niri.service`). The `systemd` package here includes `niri.service.d/session-target.conf` which wires niri into `niri-session.target`, which in turn starts DMS.

## Screenshots

Screenshots save to `~/Pictures/Screenshots/` — create this directory if it doesn't exist:

```sh
mkdir -p ~/Pictures/Screenshots
```

## prefer-no-csd

`prefer-no-csd` tells applications to use server-side decorations where possible. Apps that insist on CSD will still use it, but most GTK4/libadwaita apps will respect this.
