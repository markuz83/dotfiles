# hypr

Hyprland, hypridle, and hyprviz configuration.

## Dependencies

- `hyprland`
- `hypridle` — idle daemon
- `hyprlock` — lock screen (called by hypridle)
- `brightnessctl` — backlight control (used by hypridle)
- `dbus` — required for session activation
- DMS (DankMaterialShell) — provides `dms ipc lock lock`

## Important: most config lives in `dms/`

`hyprland.conf` sources `./dms/colors.conf`, `./dms/outputs.conf`, `./dms/binds.conf`, etc. These files are **auto-generated by DMS** and live at `~/.config/hypr/dms/`. They are not tracked in this repo — DMS regenerates them.

Only `hyprland.conf`, `hypridle.conf`, `hyprviz.conf`, and the `hyprviz/` directory are manually maintained and tracked here.

## Session startup

`hyprland.conf` starts the session with:
```
exec-once = dbus-update-activation-environment --systemd WAYLAND_DISPLAY XDG_CURRENT_DESKTOP XDG_SESSION_DESKTOP && systemctl --user start hyprland-session.target
```
This propagates Wayland environment variables to systemd and D-Bus before starting the session target. Without `dbus-update-activation-environment`, D-Bus-activated services (portals, notifications) won't have `WAYLAND_DISPLAY` set.

## hypridle timeouts

| Time | Action |
|------|--------|
| 2.5 min | Dim backlight to 10% |
| 2.5 min | Turn off keyboard backlight |
| 5 min | Lock screen |
| 5.5 min | Turn off display |
| 30 min | Suspend |

Comment out the keyboard backlight listener if your keyboard has no backlight.

## HDR

`logout.sh` resets DP-2 from HDR to SDR before exiting Hyprland. This prevents the display from being stuck in HDR after logout. The monitor name and mode are hardcoded for this machine — adjust in `logout.sh` if your setup differs.
